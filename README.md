To build and deploy a Java web application to Apache Tomcat using Jenkins Declarative Pipeline, you can integrate tools like Maven for building the project and then use SSH or the Tomcat Manager to deploy the WAR file. Below is a basic example demonstrating how to achieve this:

### Example Jenkins Declarative Pipeline

```groovy
pipeline {
    agent any
    
    environment {
        // Define environment variables as needed
        TOMCAT_HOST = 'your-tomcat-host'
        TOMCAT_PORT = '8080'
        TOMCAT_USER = credentials('tomcat-username')
        TOMCAT_PASS = credentials('tomcat-password')
    }
    
    stages {
        stage('Build') {
            steps {
                // Example: Maven build
                sh 'mvn clean package'
            }
        }
        
        stage('Deploy to Tomcat') {
            steps {
                script {
                    def warFile = findFiles(glob: 'target/*.war')[0]
                    
                    if (warFile != null) {
                        def tomcatUrl = "http://${TOMCAT_USER}:${TOMCAT_PASS}@${TOMCAT_HOST}:${TOMCAT_PORT}/manager/text"
                        
                        // Deploy WAR file to Tomcat using the Manager API
                        sh "curl --upload-file ${warFile} ${tomcatUrl}/deploy?path=/myapp&update=true"
                    } else {
                        error 'WAR file not found in target directory!'
                    }
                }
            }
        }
    }
    
    post {
        success {
            echo 'Build and deploy to Tomcat succeeded!'
        }
        failure {
            echo 'Build or deploy to Tomcat failed!'
        }
    }
}
```

### Explanation:

1. **Agent**: The pipeline runs on any available agent (`agent any`).

2. **Environment**: Defines environment variables for Tomcat host, port, username, and password. Replace `your-tomcat-host`, `8080`, `tomcat-username`, and `tomcat-password` with your actual Tomcat details. Ensure that credentials are securely managed in Jenkins.

3. **Stages**:
   - **Build**: Executes the Maven build (`mvn clean package`). Replace with the build command appropriate for your project (e.g., `gradle build` for Gradle projects).
   
   - **Deploy to Tomcat**: Uses a script block to find the WAR file generated by the build (`target/*.war`). Adjust the file pattern if your build produces a different artifact file or directory structure.

     - **Tomcat Deployment**: Uses `curl` to deploy the WAR file to Tomcat using the Tomcat Manager API (`/manager/text`). Adjust the URL path (`/myapp`) and any additional parameters (`update=true`) based on your Tomcat deployment configuration.

4. **Post Build**:
   - **Success/Failure**: Provides optional steps to execute after the pipeline completes, based on whether the pipeline succeeds or fails.

### Notes:

- Ensure that your Tomcat server is configured with the Tomcat Manager and that the Manager's username and password are correctly set in the Jenkins credentials.
- Adjust the `curl` command parameters (`--upload-file`, `${tomcatUrl}`, etc.) based on your specific deployment needs and security configurations.
- Use Jenkins credentials (`credentials('credential-id')`) to securely manage sensitive information like Tomcat username and password.
